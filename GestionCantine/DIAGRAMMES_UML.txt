DIAGRAMME DE FLUX - ENREGISTREMENT D'ACHAT
═══════════════════════════════════════════════════════════════════════════════

START
  │
  ├─► FenetreAchat.effectuerAchat()
  │     │
  │     ├─► Sélectionner élève (dropdown)
  │     │
  │     ├─► Sélectionner repas (dropdown)
  │     │
  │     └─► Cliquer "Acheter"
  │           │
  │           ├─► Cantine.enregistrerAchat(eleve, repas)
  │           │     │
  │           │     ├─► Repas.diminuerQuantite(1)
  │           │     │     └─► Vérifier quantité >= 1 ?
  │           │     │           │ OUI
  │           │     │           ├─► quantité--
  │           │     │           │
  │           │     │           │ NON
  │           │     │           └─► return false
  │           │     │
  │           │     ├─► Eleve.debiter(prix)
  │           │     │     └─► Vérifier solde >= prix ?
  │           │     │           │ OUI
  │           │     │           ├─► solde -= prix
  │           │     │           │
  │           │     │           │ NON
  │           │     │           └─► return false
  │           │     │
  │           │     ├─► new Achat(eleve, repas)
  │           │     │
  │           │     ├─► historique.add(achat)
  │           │     │
  │           │     ├─► logger.log(INFO, "Achat enregistré")
  │           │     │
  │           │     └─► return true
  │           │
  │           ├─► Afficher message "Succès" ou "Erreur"
  │           │
  │           └─► updateHistorique()
  │                 └─► Rafraîchir l'affichage
  │
  └─► END

RELATIONS ENTRE CLASSES
═══════════════════════════════════════════════════════════════════════════════

ASSOCIATION "HAS-A":
  Cantine HAS-A List<Eleve>      (1 cantine → plusieurs élèves)
  Cantine HAS-A List<Repas>      (1 cantine → plusieurs repas)
  Cantine HAS-A List<Achat>      (1 cantine → plusieurs achats)
  Achat HAS-A Eleve              (1 achat → 1 élève)
  Achat HAS-A Repas              (1 achat → 1 repas)

UTILISATION "USES-A":
  Application USES Cantine       (crée et gère l'instance)
  FenetreEleves USES Cantine     (accède aux données)
  FenetreRepas USES Cantine      (accède aux données)
  FenetreAchat USES Cantine      (effectue les opérations)
  CanteineGUI USES FenetreXXX    (ouvre les sous-fenêtres)

HÉRITAGE "IS-A":
  CanteineGUI IS-A JFrame        (extends)
  FenetreEleves IS-A JFrame      (extends)
  FenetreRepas IS-A JFrame       (extends)
  FenetreAchat IS-A JFrame       (extends)

═══════════════════════════════════════════════════════════════════════════════
ABSTRACTION ET ENCAPSULATION
═══════════════════════════════════════════════════════════════════════════════

PUBLIC INTERFACE (Ce que voit l'utilisateur):
  Cantine:
    - ajouterEleve(Eleve)
    - ajouterRepas(Repas)
    - enregistrerAchat(Eleve, Repas)
    - crediterEleve(matricule, montant)
    - obtenirEleve(matricule)

  Eleve:
    - getNom(), getPrenom(), getMatricule()
    - getSolde(), setSolde()
    - debiter(montant), crediter(montant)

  Repas:
    - getNom(), getPrix(), getQuantiteDisponible()
    - diminuerQuantite(quantite)

PRIVATE DETAILS (Cachés à l'utilisateur):
  Cantine:
    - List<Eleve> eleves
    - List<Repas> repas
    - List<Achat> historique

  Eleve:
    - String nom, prenom, matricule
    - double solde

  Repas:
    - String nom
    - double prix
    - int quantiteDisponible
