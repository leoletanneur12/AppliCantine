🏗️ ARCHITECTURE POO APPLIQUÉE
═══════════════════════════════════════════════════════════════

✅ ENCAPSULATION
   - Attributs privés dans chaque classe
   - Getters/setters publics pour accès contrôlé
   - Validation dans les setters

✅ HÉRITAGE
   - Surcharge de toString() dans toutes les classes
   - Méthodes abstraites prêtes pour extension

✅ POLYMORPHISME
   - toString() personnalisé par classe
   - Interfaces de gestion cohérentes

✅ ABSTRACTION
   - Méthodes publiques simples pour utilisateur
   - Logique complexe cachée dans la classe

✅ RESPONSABILITÉ UNIQUE
   - Chaque classe a une seule raison de changer
   - Cantine = orchestration seulement
   - Eleve = gestion élève seulement

🔧 POUR DÉMARRER L'APPLICATION
═══════════════════════════════════════════════════════════════

OPTION 1 - Windows (Plus simple):
  Double-cliquer sur: build.bat

OPTION 2 - Linux/Mac:
  chmod +x build.sh
  ./build.sh

OPTION 3 - Avec Maven:
  mvn clean compile
  mvn exec:java -Dexec.mainClass="com.cantine.Application"

OPTION 4 - Manuel:
  mkdir bin
  javac -d bin -sourcepath src/main/java src/main/java/com/cantine/*.java \
                                         src/main/java/com/cantine/model/*.java \
                                         src/main/java/com/cantine/ui/*.java
  java -cp bin com.cantine.Application

═══════════════════════════════════════════════════════════════
💡 POINTS À EXPLIQUER EN PRÉSENTATION
═══════════════════════════════════════════════════════════════

1. Architecture MVC:
   "Model" = com.cantine.model (classes métier)
   "View" = com.cantine.ui (interfaces graphiques)
   "Controller" = Application.java (orchestration)

2. Pourquoi 100% POO:
   - Chaque classe représente un concept
   - Utilisation d'héritage et polymorphisme
   - Encapsulation stricte des données
   - Responsabilité unique par classe

3. Système de logs:
   "Chaque action importante est enregistrée"
   "Permet de déboguer et d'auditer"
   "Fichier + console"

4. Flux d'un achat:
   1. Sélectionner élève et repas
   2. Vérifier solde >= prix
   3. Vérifier quantité >= 1
   4. Débiter solde
   5. Réduire quantité
   6. Enregistrer dans historique
   7. Logger l'opération
